# Adding Your Marketplace to Codex

This guide walks you through adding your plugin marketplace to [OpenAI Codex](https://developers.openai.com/codex/plugins/build).

## Prerequisites

- Codex installed and running
- Your marketplace repository or directory ready

## Your marketplace structure

After running `pmw start` and saving with the **Codex** export target enabled, your marketplace directory should look like this:

```
my-marketplace/
тФЬтФАтФА .agents/
тФВ   тФФтФАтФА plugins/
тФВ       тФФтФАтФА marketplace.json
тФЬтФАтФА plugins/
тФВ   тФФтФАтФА my-plugin/
тФВ       тФЬтФАтФА .codex-plugin/
тФВ       тФВ   тФФтФАтФА plugin.json
тФВ       тФЬтФАтФА .mcp.json
тФВ       тФЬтФАтФА skills/
тФВ       тФВ   тФФтФАтФА my-skill/
тФВ       тФВ       тФФтФАтФА SKILL.md
тФВ       тФФтФАтФА agents/
тФВ           тФФтФАтФА my-agent.md
```

Key differences from other platforms:

- The root marketplace manifest lives at `.agents/plugins/marketplace.json`
- Each plugin has its `plugin.json` inside a `.codex-plugin/` folder
- The marketplace manifest uses a structured `source` object with `source` and `path` fields
- Each plugin entry includes a `policy` object controlling installation behavior

## Step 1: Set up the marketplace file

The marketplace file at `.agents/plugins/marketplace.json` is generated automatically by Plugin Marketplace Wizard. It looks like this:

```json
{
  "name": "my-marketplace",
  "interface": {
    "displayName": "My Marketplace"
  },
  "plugins": [
    {
      "name": "my-plugin",
      "source": {
        "source": "local",
        "path": "./plugins/my-plugin"
      },
      "policy": {
        "installation": "AVAILABLE",
        "authentication": "ON_INSTALL"
      },
      "category": "Productivity"
    }
  ]
}
```

## Step 2: Deploy as a repo marketplace

Copy the marketplace into your repo root. The `.agents/plugins/marketplace.json` file should be at the repository root level:

```shell
# Your repo should contain:
# .agents/plugins/marketplace.json
# plugins/<plugin-slug>/...
```

## Step 3: Restart Codex

After adding or updating the marketplace file, restart Codex so it picks up the new configuration.

## Step 4: Browse and install plugins

Open the plugin directory in Codex, choose your marketplace, and browse or install the plugins in your curated list.

## Using the `$plugin-creator` skill

Codex includes a built-in `$plugin-creator` skill that can scaffold the plugin manifest and generate a marketplace entry. This is an alternative to using Plugin Marketplace Wizard for Codex-only marketplaces.

## Plugin manifest details

The `.codex-plugin/plugin.json` generated by Marketplace Wizard includes:

```json
{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "A useful plugin",
  "author": {
    "name": "Your Name",
    "email": "you@example.com"
  },
  "skills": "./skills/",
  "mcpServers": "./.mcp.json"
}
```

For richer install-surface metadata, you can manually add an `interface` block with `displayName`, `shortDescription`, `brandColor`, `logo`, and other fields.

## Updating the marketplace

After editing plugins, update the marketplace directory and restart Codex so the local install picks up the new files.

## Personal marketplace

For a personal (non-repo) marketplace, use `~/.agents/plugins/marketplace.json` and store plugins under `~/.codex/plugins/`.
