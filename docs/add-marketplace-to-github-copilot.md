# Adding Your Marketplace to GitHub Copilot CLI

This guide walks you through adding your plugin marketplace to [GitHub Copilot CLI](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/plugins-marketplace).

## Prerequisites

- GitHub Copilot CLI installed and authenticated (`copilot --version`)
- Your marketplace repository pushed to GitHub (or any git host)

## Your marketplace structure

After running `pmw start` and saving with the **GitHub Copilot** export target enabled, your marketplace directory should look like this:

```
my-marketplace/
тФЬтФАтФА .github/
тФВ   тФФтФАтФА plugin/
тФВ       тФФтФАтФА marketplace.json
тФЬтФАтФА plugins/
тФВ   тФФтФАтФА my-plugin/
тФВ       тФЬтФАтФА plugin.json
тФВ       тФЬтФАтФА .mcp.json
тФВ       тФЬтФАтФА skills/
тФВ       тФВ   тФФтФАтФА my-skill/
тФВ       тФВ       тФФтФАтФА SKILL.md
тФВ       тФФтФАтФА agents/
тФВ           тФФтФАтФА my-agent.md
```

Key differences from Cursor and Claude:

- The root marketplace manifest lives at `.github/plugin/marketplace.json`
- Each plugin has its `plugin.json` at the **root** of the plugin directory (not inside a hidden folder)
- The `plugin.json` includes component directory references (`skills`, `agents`, `mcpServers`)

Push this to a GitHub repository (e.g. `your-org/my-marketplace`).

## Step 1: Add the marketplace

Open your terminal and run:

```shell
copilot plugin marketplace add your-org/my-marketplace
```

This registers the marketplace from your GitHub repository. Copilot CLI reads the `.github/plugin/marketplace.json` file to discover available plugins.

### Other source types

You can also add from other sources:

```shell
# From any git URL
copilot plugin marketplace add https://gitlab.com/your-org/my-marketplace.git

# From a local directory (great for testing)
copilot plugin marketplace add ./my-marketplace
```

## Step 2: Browse available plugins

List all plugins available from your registered marketplaces:

```shell
copilot plugin list --available
```

Or in an interactive session:

```copilot
/plugin list
```

## Step 3: Install a plugin

Install a plugin from your marketplace:

```shell
copilot plugin install my-plugin
```

After installation, verify it loaded correctly:

```shell
copilot plugin list
```

## Step 4: Verify components loaded

Start a new interactive session and check that the plugin's components are available:

```copilot
# Check agents
/agent

# Check skills
/skills list
```

## Updating the marketplace

Refresh your marketplace to pull the latest plugins:

```shell
copilot plugin marketplace update your-org/my-marketplace
```

## Plugin manifest details

The `plugin.json` generated by Marketplace Wizard includes references to the plugin's components:

```json
{
  "name": "my-plugin",
  "description": "A useful plugin",
  "version": "1.0.0",
  "author": {
    "name": "Your Name",
    "email": "you@example.com"
  },
  "skills": "skills/",
  "agents": "agents/",
  "mcpServers": ".mcp.json"
}
```

These fields tell Copilot CLI where to find each component type within the plugin directory.

## Testing locally

Before pushing to GitHub, test your marketplace locally:

```shell
copilot plugin install ./my-marketplace/plugins/my-plugin
copilot plugin list
```

To pick up changes after editing a local plugin, reinstall it:

```shell
copilot plugin install ./my-marketplace/plugins/my-plugin
```

## Removing a plugin or marketplace

```shell
# Uninstall a single plugin
copilot plugin uninstall my-plugin

# Remove the entire marketplace
copilot plugin marketplace remove your-org/my-marketplace
```
